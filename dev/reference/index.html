ObliviousOffload.access_log_middleware — Method
access_log_middleware(handler)

Log each request in an nginx-like access log format:

127.0.0.1 - [14/Jul/2026:13:37:00 +0000] "POST /endpoint HTTP/1.1" 200 1234 0.042s

ObliviousOffload.load_config — Method
load_config() -> NamedTuple

Load the connection configuration from LocalPreferences.toml (section [ObliviousOffload]). Recognized keys: port, hostname, username, password. Missing keys fall back to defaults; username and password default to nothing, which disables basic auth.

hostname is the server's public name: it is placed in the TLS certificate's SAN and used by clients as the address to connect to. The server itself always listens on all interfaces (0.0.0.0).

ObliviousOffload.make_part — Method
make_part(obj) -> HTTP.Multipart

Serialize obj into an HTTP.Multipart part using Julia's Serialization stdlib.

The content type application/x-julia-serialized-object follows the convention established by Java's application/x-java-serialized-object for language-specific serialized objects.

source
ObliviousOffload.parse_parts — Method
parse_parts(parts::Vector{HTTP.Multipart}) -> Dict{String, Any}

Deserialize a vector of multipart form parts into a name-value dictionary. Parts with content type application/x-julia-serialized-object are deserialized via Serialization.deserialize.
